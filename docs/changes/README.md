# Change Index

I use this folder to record why I changed the repo, not just what changed.

This matters because I do not want each release or maintenance pass to restart the same thinking from zero.

Use:

- `*.prd.md` for what the change is trying to achieve and why it matters
- `*.decision.md` for tradeoffs, sequencing, and final choices
- `*.spec.md` for the operating rules future work should follow
- `*.logs.md` only when failures or lessons are worth preserving

## Naming Convention

Use date-prefixed filenames so changes sort naturally:

```text
YYYY-MM-DD-change-name.prd.md
YYYY-MM-DD-change-name.decision.md
YYYY-MM-DD-change-name.spec.md
YYYY-MM-DD-change-name.logs.md
```
