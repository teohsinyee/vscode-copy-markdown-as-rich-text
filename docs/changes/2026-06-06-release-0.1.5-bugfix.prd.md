# PRD: Release 0.1.5 Bug Fix

## Goal

Ship a safe `0.1.5` bug-fix release for current users without changing the extension's core workflow.

## Why This Matters

This extension already has active users and sits on a fragile copy pipeline. A careless bug fix can easily damage rich-text output, clipboard fallback behavior, or Windows-specific handling.

This release should improve reliability without turning into a refactor project.

## Problems In Scope

- Rich-text output truncates longer content in target editors
- Numbered-list indentation and layout are not preserved after paste
- Ampersands in headings render as HTML entities in pasted output
- Code blocks lose alignment and line structure after paste
- Comparison operators and escaped characters render incorrectly in pasted output

## Release Scope

- Fix selected rich-text copy bugs
- Preserve the current command flow
- Preserve plain-text fallback behavior
- Keep changes scoped to bug-fix work only

## Out Of Scope

- New features
- UI redesign
- Dependency refresh for unrelated packages
- Broad cleanup or refactor work
- Marketplace content changes unless required by the bug-fix release itself

## Success Criteria

- Existing users can keep using the command the same way they do today
- Fixed bugs are covered by appropriate validation
- Windows clipboard behavior does not regress
- The release process is documented well enough to reuse later
